## 2. Методология описания

### 2.1 Принцип
Структура данных описывает **контракты хранения и обмена данными** в текущей модели `repo + scripts + GitHub Actions`, без отдельной прикладной СУБД на MVP.

### 2.2 Выбор модели представления
- модель: `гибридная (hybrid)`
- единицы описания: `документ (document)`, `хранилище объектов (object-storage)`, `журнал событий/поток (event-log/stream)`, `kv-пространство (kv)`
- основание: `docs/requirements/обоснование выбора.md` (облачная-first модель GitHub Actions (Cloud-first), источник истины (source of truth) в репозитории, артефакты/логи (artifacts/logs) в CI)

### 2.3 Формат диаграмм
- формат диаграмм: `plantuml`
- основание: `docs/requirements/обоснование выбора.md` (раздел 1.3)

## 6. Связи и жизненный цикл данных
- `policy-manifests` -> определяют `check-plan-snapshot`.
- `repository-integration-profiles` -> задают `enforcement_mode` и `rollout_channel` для конкретного репозитория.
- `check-plan-snapshot` -> управляет генерацией `check-run-events`.
- `check-run-events` -> порождают `verification-artifacts` и `run-summaries`.
- `run-summaries` и `verification-artifacts` используются как evidence для PR и аудита.
- `ci-cache` не является источником истины (source of truth) и может очищаться без потери бизнес-данных.
