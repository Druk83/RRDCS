## 1. Конвенции

### 1.1 Классификация событий
- **Domain Event:** факт управления policy/toolchain внутри governance-домена.
- **Integration Event:** публикация policy и baseline в другие домены.

### 1.2 Общая оболочка события
- `eventId`, `eventType`, `eventVersion`, `occurredAt`, `correlationId`, `causationId`, `producer`, `payload`

### 1.3 Идемпотентность и доставка
- **Delivery:** `at-least-once`
- **Idempotency key:** `eventId`

## 4. Карточки событий

### 4.1 EVT: GovernanceDecisionRecorded
- **Тип:** Domain
- **Описание:** зафиксировано решение governance по изменению policy/toolchain.
- **Когда публикуется:** после review и проверки policy/pinned versions.
- **Кто публикует:** BC Управление политиками и цепочкой инструментов (Policy and Toolchain Governance BC)
- **Кто потребляет:** Git-репозиторий и поток PR (Git Repository and Pull Request Flow), BC Оркестрация проверок качества (Quality Gate Orchestration BC), BC Пакеты проверок (Verification Packages BC)
- **Команда/политика-источник:** CMD `ReviewPolicyChange`, POL `If policy valid and approved then publish new policy version`
- **Связанные события:** `PolicyValidationCompleted`, `PolicyPublishedToOrchestrator`, `PolicyChangeRejected`

#### 4.1.1 Payload (поля)
| Поле | Тип | Обязательность | Описание | Пример |
|---|---|---:|---|---|
| decisionId | string | yes | Идентификатор governance-решения | "gov-20260403-05" |
| policyId | string | yes | Идентификатор policy | "quality-policy-v3" |
| policyChangeId | string | yes | Идентификатор изменения policy | "chg-142" |
| decision | string | yes | Решение (`approved|rejected`) | "approved" |
| approvedBy | string | yes | Ответственный reviewer | "tech-lead-01" |
| requiredChecks | array | yes | Итоговый список required checks | ["shellcheck", "codeql", "tests"] |
| pinnedVersions | object | yes | Зафиксированные версии runtime/toolchain | {"python":"3.12.2","node":"22.8.0"} |
| decidedAt | string (ISO-8601) | yes | Время решения | "2026-04-03T12:10:00Z" |

```json
{

  }
}
```

### 4.1.3 Семантика и гарантии
- **Идемпотентность:** по `eventId`.
- **Delivery:** `at-least-once`.
- **Порядок:** `per-key` по `policyChangeId`.
- **Совместимость версий:** policy schema изменяется через versioned commits.
- **Ошибки и ретраи:** retry публикации в subscribers; при fail фиксируется governance incident.
- **SLA:** `P1`.
