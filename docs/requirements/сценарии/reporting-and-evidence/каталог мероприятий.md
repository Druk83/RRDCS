## 4. Карточки событий

### 4.1 EVT: PRSummaryPublished
- **Тип:** Domain
- **Описание:** опубликован итоговый summary проверки PR с причинами падений и ссылками на evidence.
- **Когда публикуется:** после получения gate-итога и привязки логов/артефактов.
- **Кто публикует:** BC Отчетность и доказательная база (Reporting and Evidence BC)
- **Кто потребляет:** Разработчик (Developer), Технический лидер и архитектор (Tech Lead and Architect)
- **Команда/политика-источник:** CMD `PublishPRSummary`, POL `If failed checks exist then include reason and failed step reference`
- **Связанные события:** `GateRunCompleted`, `VerificationDetailsAvailable`, `EvidenceLinksPublished`

#### 4.1.1 Payload (поля)
| Поле | Тип | Обязательность | Описание | Пример |
|---|---|---:|---|---|
| prId | string | yes | Идентификатор PR | "PR-184" |
| runId | string | yes | Идентификатор run | "run-20260403-184" |
| overallStatus | string | yes | Итог summary (`passed|failed`) | "failed" |
| failedChecks | array | yes | Список упавших checks | ["codeql"] |
| failReasons | array | yes | Причины падений по checks | ["codeql: high severity alert"] |
| evidenceLinks | array | yes | Ссылки на логи/артефакты | ["https://.../artifact/1"] |
| publishedAt | string (ISO-8601) | yes | Время публикации summary | "2026-04-03T12:25:00Z" |

#### 4.1.2 Пример события (JSON)
```json
{
  "eventId": "evt-re-0009",
  "eventType": "PRSummaryPublished",
  "eventVersion": 1,
  "occurredAt": "2026-04-03T12:25:00Z",
  "correlationId": "run-20260403-184",
  "causationId": "exec-20260403-001",
  "producer": "reporting-and-evidence-bc",
  "payload": {
    "prId": "PR-184",
    "runId": "run-20260403-184",
    "overallStatus": "failed",
    "failedChecks": ["codeql"],
    "failReasons": ["codeql: high severity alert"],
    "evidenceLinks": ["https://ci.example/artifacts/run-20260403-184/codeql.log"],
    "publishedAt": "2026-04-03T12:25:00Z"
  }
}
```

### 4.1.3 Семантика и гарантии
- **Идемпотентность:** по `eventId`.
- **Delivery:** `at-least-once`.
- **Порядок:** `per-key` по `runId`.
- **Совместимость версий:** изменения payload контролируются `eventVersion`.
- **Ошибки и ретраи:** повторная публикация summary до успешного подтверждения PR UI.
- **SLA:** `P1`.
