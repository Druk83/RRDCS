## 4. Карточки событий

### 4.1 EVT: SecurityChecksCompleted
- **Тип:** Domain
- **Описание:** завершен security baseline, сформированы findings и итоговый статус.
- **Когда публикуется:** после выполнения security-инструментов в рамках required check-набора.
- **Кто публикует:** BC Пакеты проверок (Verification Packages BC)
- **Кто потребляет:** BC Оркестрация проверок качества (Quality Gate Orchestration BC), Reporting and Evidence
- **Команда/политика-источник:** CMD `RunSecurityChecks`, POL `If blocking findings exist then mark set failed`
- **Связанные события:** `BlockingSecurityFindingDetected`, `VerificationSetPublished`

#### 4.1.1 Payload (поля)
| Поле | Тип | Обязательность | Описание | Пример |
|---|---|---:|---|---|
| runId | string | yes | Идентификатор запуска проверок | "run-20260403-184" |
| status | string | yes | Итог security phase (`passed|failed`) | "failed" |
| findingsCount | integer | yes | Количество найденных issues | 2 |
| maxSeverity | string | yes | Максимальная severity finding | "high" |
| blockingFindings | integer | yes | Количество блокирующих findings | 1 |
| tools | array | yes | Использованные инструменты | ["gitleaks", "codeql"] |
| completedAt | string (ISO-8601) | yes | Время завершения фазы | "2026-04-03T11:02:00Z" |

#### 4.1.2 Пример события (JSON)
```json
{
  "eventId": "evt-vp-0007",
  "eventType": "SecurityChecksCompleted",
  "eventVersion": 1,
  "occurredAt": "2026-04-03T11:02:00Z",
  "correlationId": "run-20260403-184",
  "causationId": "exec-20260403-001",
  "producer": "verification-packages-bc",
  "payload": {
    "runId": "run-20260403-184",
    "status": "failed",
    "findingsCount": 2,
    "maxSeverity": "high",
    "blockingFindings": 1,
    "tools": ["gitleaks", "codeql"],
    "completedAt": "2026-04-03T11:02:00Z"
  }
}
```

### 4.1.3 Семантика и гарантии
- **Идемпотентность:** по `eventId`.
- **Delivery:** `at-least-once`.
- **Порядок:** `per-key` по `runId`.
- **Совместимость версий:** расширение payload допускается без смены major версии.
- **Ошибки и ретраи:** retry публикации; fail логируется с `runId`.
- **SLA:** `P1`.
