## 1. Конвенции

### 1.1 Классификация событий
- **Domain Event:** факт внутри orchestration-домена.
- **Integration Event:** событие обмена между доменами или внешними системами.
- **Technical Event:** не используется как основной контракт в этом каталоге.

### 1.2 Общая оболочка события

- **Idempotency key:** `eventId`

## 4. Карточки событий

### 4.1 EVT: MergeDecisionRecorded
- **Тип:** Domain
- **Описание:** зафиксировано итоговое решение о допуске или блокировке merge для PR.
- **Когда публикуется:** после получения полного результата required checks и применения policy.
- **Кто публикует:** BC Оркестрация проверок качества (Quality Gate Orchestration BC)
- **Кто потребляет:** API pull request в GitHub (GitHub Pull Request API), Reporting and Evidence
- **Команда/политика-источник:** CMD `PublishGateOutcome`, POL `If all required checks passed then allow merge else block merge`
- **Связанные события:** `RequiredVerificationSetCompleted`, `RequiredCheckFailed`

#### 4.1.1 Payload (поля)
| Поле | Тип | Обязательность | Описание | Пример |
|---|---|---:|---|---|
| prId | string | yes | Идентификатор PR | "PR-184" |
| executionId | string | yes | Идентификатор orchestration-run | "exec-20260403-001" |
| decision | string | yes | Итог: `allowed` или `blocked` | "blocked" |
| requiredChecksPassed | boolean | yes | Признак прохождения required checks | false |
| failedChecks | array | yes | Список упавших checks | ["shellcheck", "codeql"] |
| reason | string | yes | Причина блокировки/допуска | "required checks failed" |
| enforcementMode | string | yes | Режим применения (`audit|required`) | "required" |
| decidedAt | string (ISO-8601) | yes | Время фиксации решения | "2026-04-03T10:35:00Z" |

#### 4.1.2 Пример события (JSON)
```json
{
  "eventId": "evt-qgo-0001",
  "eventType": "MergeDecisionRecorded",
  "eventVersion": 1,
  "occurredAt": "2026-04-03T10:35:00Z",
  "correlationId": "PR-184",
  "causationId": "exec-20260403-001",
  "producer": "quality-gate-orchestration-bc",
  "payload": {
    "prId": "PR-184",
    "executionId": "exec-20260403-001",
    "decision": "blocked",
    "requiredChecksPassed": false,
    "failedChecks": ["shellcheck", "codeql"],
    "reason": "required checks failed",
    "enforcementMode": "required",
    "decidedAt": "2026-04-03T10:35:00Z"
  }
}
```

### 4.1.3 Семантика и гарантии
- **Идемпотентность:** по `eventId`.
- **Delivery:** `at-least-once`.
- **Порядок:** `per-key` по `prId`.
- **Совместимость версий:** `eventVersion` увеличивается при несовместимых изменениях payload.
- **Ошибки и ретраи:** retry публикации до 3 попыток; при исчерпании фиксируется fail-лог.
- **SLA:** `P1`.
