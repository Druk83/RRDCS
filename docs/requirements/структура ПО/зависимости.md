## 1. Сводка
- Инструменты RRDCS реализованы на Python Standard Library (без обязательных `pip`-зависимостей).
- Введены пакеты проверок `style/security/platform/architecture` с единым контрактом запуска.
- Нормативный style toolchain для этого репозитория: `ShellCheck`, `Hadolint`, `markdownlint`, `actionlint`; `ClangTidy` не входит в текущий стек.
- Версии внешних quality/security инструментов фиксируются в policy-файле `.sources/.manifest/policies/check-packages.yaml`.
- CI-оркестрация выполняется в GitHub Actions с pinned versions (`actions/checkout`, `actions/setup-python`, `actions/upload-artifact`).
- Для локального рендера диаграмм сохраняется инфраструктурная зависимость `yuzutech/kroki:0.30.1`.

## 3. Политика управления версиями
- Базовые внешние инструменты quality/security фиксируются в `.sources/.manifest/policies/check-packages.yaml`.
- Обновление версий выполняется по циклу `monthly`.
- Стратегия pinning: `pin-major-minor`.
- Любое обновление проходит проверку:
  - локальный запуск quality-gates пакетов;
  - запуск PR workflow в `linux + windows`.

## 4. Риски и меры

| Риск | Уровень | Мера |
|---|---|---|
| Отсутствие tool binaries в локальной среде разработчика | MEDIUM | Использовать режим `--allow-missing-tools` для локальной диагностики; в CI инструменты должны быть обязательными |
| Drift версий external tools в разных репозиториях | MEDIUM | Централизовать версии через policy-файл `check-packages.yaml` |
| Недоступность внешнего `kroki.io` | MEDIUM | Использовать локальный Kroki через Docker |
