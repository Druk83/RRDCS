## 1. Сводка
- Прямые package-менеджер зависимости уровня приложения (`npm`, `pip`, `cargo`, `go`) в репозитории сейчас не зафиксированы.
- Оба локальных инструмента `.tools/*` используют стандартную библиотеку Python (Python Standard Library).
- Инфраструктурная зависимость для локального рендера диаграмм: Docker image `yuzutech/kroki:0.30.1`.
- Основной риск зависимости: отсутствие зафиксированных CI action versions, потому что `.github/workflows/*` еще не реализованы.

## 5. Политика управления зависимостями
- Для `.tools/*` сохраняется курс на stdlib-only подход, чтобы минимизировать внешние пакеты.
- Для Docker image используется фиксированный tag, обновление выполняется осознанно после проверки совместимости.
- При добавлении CI workflows обязательно фиксировать версии GitHub Actions (pin by version, для критичных — pin by commit SHA).
- Любая новая внешняя зависимость должна проходить проверку по `.manifest/dependenciesmanifest.md`.

## 6. Риски и меры
